Small packages for building browser apps with clear runtime contracts.

Why AI-friendly

Built for agents that need reliable boundaries.

Package-local contracts

Each published package keeps a focused public API and local tests, so generated changes can stay inside the right boundary.

Plain browser runtime

Apps can run as static files with browser storage and standard DOM APIs, reducing deployment assumptions.

Explicit data flow

Observable state, event APIs, bindings, and local tables make state changes inspectable and easy to reason about.

Useful together or separately

Use the full stack for generated local-first apps, or pull in a lower-level package when a focused utility is enough.
