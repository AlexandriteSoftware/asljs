Read list of files from "files" property of the envelope JSON.

Implement user's request by one of the following:

- if problem is clear, and the patch is provided but the user reported issues
  with the patch: recreate the patch from the envelope, do not create
  incremental patches
- if problem is clear, there is enough data, and solution is ready:
  create patch that modifies files and envelope
- if problem is clear, but there is not enough data:
  - create patch that reads files and ask user to apply it it and provide new
    envelope
  - ask user to run a COG command, powershell command or script and provide
    the result

Envelope format:

{
  "instruction": "...",
  "files": [
    {
      "path": "path/to/file",
      "type": "text",
      "content": "...",
      "complete": true,
      "update": {
        ... can be ignored for the purpose of this instruction ...
      }
    }
  ]
}

A patch is a JSON file containing a task and a list of commands.

Patch format:
{
  "task": "...",
  "commands": [
    {
      "command": "read",
      "pattern": "path/to/file",
      "exclude": [],
      "lines": 100,
      "sizeKb": 20
    },
    { "command": "write", "path": "path/to/file", "content": "..." },
    { "command": "remove", "path": "path/to/file" }
  ]
}

Available patch commands:
- read: reads matching files and adds them to the envelope.
- write: creates a file or sets the content of a file.
- remove: removes a file.

CLI commands:
- `cog read <path> [arguments]` reads files into the envelope.
- `cog update` refreshes envelope files using their stored update commands.
- `cog restore` restores files from `backup.json` and removes the backup file.
- `cog apply-patch` applies the patch transactionally to the project files and
  updates the envelope.
